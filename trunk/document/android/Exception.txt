[解决]
初学者需要多加注意，新的android SDK1.5提供了avd，可以更好的模拟真实环境。但是需要设置ANDROID_SDK_HOME环境变量，
否则avd manger和android命令默认都会把新建的avd放到用户目录下，但emulator默认却不会到用户目录搜索。
设置ANDROID_SDK_HOME后，大家都会到该目录下搜索。初学者需要多加注意
Android_SDK_HOME路径是你解压下载的SDK路径， 在path环境变量里配置%Android_SDK_HOME%\tools,


android create avd -n my_android1.5 -t 2
emulator -avd my_android1.5 -sdcard F:\mysdcard.img

解决：
1、
对于ERROR/libEGL(259): couldn't load <libhgl.so> library (Cannot load library: load_library[984]: Library 'libhgl.so' not found)
解析如下：
this is by no means a defect. the system simply tries to load the hardware opengl driver and fails. 
it then falls back to the software implementation. this is normal for all opengl apps on android and is not related to libgdx in any way
摘自：http://code.google.com/p/libgdx/issues/detail?id=30

2、WARN/dalvikvm(259): No implementation found for native Lorg/tinder/studio/gl/HelloGLView$HelloGLRender;.nativeInit ()

    }
